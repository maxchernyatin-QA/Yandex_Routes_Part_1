В рамках проекта была поставлена цель протестировать валидацию полей ввода времени и адресов, логику расчёта стоимости и времени поездки на собственном автомобиле в веб-приложении "Яндекс. Маршруты".

Использованные инструменты: Figma, DevTools.

Применённые виды тестирования: функциональное тестирование, UI-тестирование, тестирование веб-приложенийе.

Применённые техники тест-дизайна: классы эквивалентности, граничные значения.

Выполнено:

1. Проведен тест-анализ, выделены основные блоки в тестировании логики полей ввода.

2. Выделены классы эквивалентности и граничные значения для проведения тестирования логики полей ввода.

3. Составлено 70 тест-кейсов, выполнено тестирование, обнаружено 16 багов.

4. Составлено 16 баг-репортов.
